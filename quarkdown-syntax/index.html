Quarkdown 语法备忘录

写在前面

Quarkdown 是在 Markdown 之上扩展而来的，文件扩展名是 .qd。

大多数标准 Markdown 语法（标题、列表、引用、代码块、链接、图片）都直接可用。除此之外，Quarkdown 增加了两类扩展：

  • 以 . 开头的函数式指令（如 .center、.align、.tableofcontents、.docname）
  • 以 ! 开头的”装饰型块”（如 #! ...，用于诗体引用、题词）

下面按”够用就行”的原则整理一份常用语法清单。完整语法请看官方文档。

元数据指令

放在文件开头，控制整篇文档的属性：

.docname {blog-home}            # 文档名（用作 URL slug）
.docdescription {主页}          # 文档描述（HTML meta）
.doctype {plain}                # 文档类型：plain | docs
.doclang {Chinese}              # 语言标识
.theme {paperwhite} layout:{latex}  # 主题与排版引擎
.docauthors
  - 清河                       # 作者列表，可多行

常用 doctype：

  • plain — 普通文档（最常用）
  • docs — API 文档风格（带侧边栏导航）

标题与段落

# 一级标题
## 二级标题
### 三级标题

段落之间用空行分隔。

列表

无序列表：

- 第一项
- 第二项
  - 嵌套项
- 第三项

有序列表：

1. 第一步
2. 第二步
3. 第三步

链接与图片

[链接文字](https://example.com)
[相对路径链接](./other-page.qd)
![图片描述](./images/foo.png)

相对路径在本地 file:// 打开和 CI 部署后访问根路径都能正常解析，推荐优先用相对路径。

引用

普通引用：

> 这是一段引用文字。
> 可以跨多行。

诗体引用（题词专用，用 #! 开头，常与 .center 配合）：

.center
    #! 最是人间留不住，朱颜辞镜花辞树
.align {end}
    -- 王国维

Quarkdown 专属扩展

居中与对齐

.center
    内容居中显示
.align {end}
    内容右对齐

自动生成目录

.tableofcontents

放在需要目录的位置即可。生成深浅由文件元数据里的 .numbering 控制。

编号格式

.numbering
    - headings: 1.1.1
    - figures: 1.1

控制标题和图注的编号风格。

主题与排版

.theme {paperwhite} layout:{latex}

常用主题：paperwhite、darko。常用排版：latex、plain。

与标准 Markdown 的差异速查

场景标准 MarkdownQuarkdown
文件扩展名.md.qd
元数据YAML frontmatter.docname {xxx} 函数式指令
诗体引用不支持#! ... 配合 .center
主题切换不支持.theme {xxx}
