JavaScript

基础语法

  • 分号 JavaScript 中分号是可选的，省略封号不算错误，为了保持一致性，建议始终使用分号。

变量

创建变量

let name;
name = "张三";
// 另外一种创建变量的方式是 let name = "张三"; 这个方式。
console.log(name);
  // 输出：张三

let 和 var 之间存在一些微小差异。

变量赋值

let harryAddress = "Hello World";
let deliveryAddress = harryAddress;
console.log(deliveryAddress);
  // 输出：Hello World

JavaScript 变量命名限制

  • 变量名称只能包含字母、数字、下划线和美元符号。
  • 变量名称必须以字母、下划线或美元符号开头，不能以数字开头。
  • 变量名称不能是 JavaScript 的保留字。

JavaScript 变量命名规则

  • 驼峰命名法：变量名称的第一个单词以小写字母开头，后续单词的首字母大写，例如：myVariableName。
  • 尽量选择有意义的变量名。

常量

常量是一个不可改变的变量。

声明方式

使用关键字 const 来声明这样的变量，设置值后，值无法再次改变。

使用情况

  1. 声明在程序编写阶段就已知值的不可改变变量。
  2. 声明在程序执行期间无需更改的变量。

常量命名规则和使用好处

  • 常量的命名规则与变量相同，建议使用全大写字母和下划线来区分常量，比如 const MAX_VALUE = 100;。 决定使用 const 还是 let 取决于变量会不会被重新赋值，而不是看这个值本身是不是字符串、路径之类。
  • 例子
    const homeDirPath = ... // 这里可能有代码用于获取此路径
    表达的意思：虽然 homeDirPath 的值不是手写死的，可能是“通过某段代码算出来的”或者“从系统里获取到的”，但只要这个变量在后面不打算再改，那就应该用 const。

使用常量的好处

  1. 提高代码的可读性：常量名称通常具有描述性。
  2. 编程更加简单：不需要记住具体的值，只需要使用常量名称。
  3. 减少更改量：如果需要更改常量的值，只需修改常量的定义，而不需要修改代码中所有使用该常量的地方。
